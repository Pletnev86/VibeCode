# Команды для чата - Доработка модулей

## ⚡ Быстрый старт

**Если кнопка "Доработать модули" не работает** - просто используй чат напрямую с командами ниже!

## Как доработать модули через чат

Вместо кнопки "Доработать модули" можно использовать чат напрямую. Просто напиши в чат одну из команд ниже:

## Команды для доработки модулей

### 1. Доработать конкретный файл
```
Доработай main.js: добавь логирование и обработку ошибок
```

```
Доработай ui.js: добавь валидацию ввода и обработку ошибок
```

```
Доработай preload.js: добавь все IPC методы
```

### 2. Доработать модуль с описанием задачи
```
Доработай модуль file-parser.js: улучши парсинг путей для Windows
```

```
Доработай chat-context.js: добавь кеширование структуры проекта
```

```
Доработай backup-manager.js: добавь автоматическую очистку старых бэкапов
```

### 3. Добавить функционал в файл
```
Добавь в main.js функцию автоматического восстановления из бэкапа при ошибке
```

```
Добавь в ui.js обработку ошибок при отправке сообщений
```

```
Добавь в preload.js метод для удаления файлов
```

### 4. Улучшить существующий код
```
Улучши main.js: добавь детальное логирование всех операций
```

```
Улучши ui.js: добавь индикатор загрузки при отправке сообщений
```

```
Улучши chat-context.js: добавь автоматическое обновление структуры проекта
```

### 5. Исправить проблемы
```
Исправь ошибки в main.js: добавь проверку существования файлов перед чтением
```

```
Исправь баги в ui.js: добавь обработку случая когда window.api не определен
```

### 6. Добавить новые функции
```
Добавь функцию автоматического тестирования в main.js
```

```
Добавь функцию экспорта логов в ui.js
```

## Примеры полных запросов

### Пример 1: Доработка с детальным описанием
```
Доработай main.js: добавь все IPC обработчики (generate-project, send-chat-message, get-logs, read-file, write-file, run-command, analyze-project, enhance-modules), логирование через logger.js, инициализацию агентов через initAgents(), базу знаний через initKnowledgeBase(). Используй рабочую версию из бэкапа как образец.
```

### Пример 2: Добавление функционала
```
Добавь в main.js функцию автоматического создания бэкапа перед каждым Self-Build и восстановления при ошибке
```

### Пример 3: Улучшение существующего кода
```
Улучши ui.js: добавь обработку ошибок, индикатор загрузки, и отображение сохраненных файлов из ответа AI
```

### Пример 4: Исправление проблем
```
Исправь проблему с кнопкой "Доработать модули": добавь обработчик события click и проверку существования window.api.enhanceModules
```

## Формат ответа AI

AI автоматически:
1. Прочитает текущий файл (если он существует)
2. Поймет что нужно изменить
3. Предоставит ПОЛНЫЙ код файла со всеми изменениями
4. Система автоматически сохранит файл

## Важно!

- Всегда указывай конкретный файл для доработки
- Описывай что именно нужно добавить/изменить
- AI автоматически видит текущее содержимое файла
- AI предоставляет ПОЛНЫЙ код файла, а не только фрагменты

## Альтернатива кнопке

Если кнопка "Доработать модули" не работает, просто используй чат с командами выше - результат будет тот же!

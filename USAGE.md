# Инструкция по использованию VibeCode

## Запуск приложения

### 1. Убедитесь, что LM Studio запущен

LM Studio должен быть запущен и слушать на порту 1234 с загруженными моделями:
- `deepseek/deepseek-r1-0528-qwen3-8b` - для reasoning, объяснений, переводов
- `nomic-ai-gpt4all-falcon` - для генерации кода (только английский)

### 2. Запуск приложения

```bash
npm start
```

Или в режиме разработки (с DevTools):

```bash
npm run dev
```

## Использование интерфейса

### Переключение AI Провайдера

В боковой панели вы можете переключиться между:
- **LM Studio (локальный)** - для использования локальных моделей
- **OpenRouter (API)** - для использования внешних моделей через API

**Где находится:** Боковая панель → секция "AI Провайдер"

**Как использовать:**
1. Выберите нужный провайдер радио-кнопкой
2. Система автоматически переключится
3. Все последующие запросы будут использовать выбранный провайдер

Подробнее см. [PROVIDER_SWITCH.md](docs/PROVIDER_SWITCH.md)

### Кнопка "Self-Build"

Нажмите кнопку "Self-Build" для автоматической генерации каркаса проекта на основе Vision.md и Roadmap.md.

Процесс:
1. Читает Vision.md и Roadmap.md
2. Формирует промпт для AI
3. Генерирует файлы через AI Router
4. Сохраняет файлы в директорию `/src`

### Кнопка "Анализ проекта"

Анализирует структуру существующего проекта (по умолчанию текущая директория).

Показывает:
- Количество директорий и файлов
- Markdown файлы
- Конфигурационные файлы
- Код файлы

### Чат-интерфейс

Используйте текстовое поле для:
- Задач по разработке
- Вопросов к AI
- Команд для генерации кода

**Примечание:** Falcon работает только с английским языком. Русские запросы автоматически переводятся через DeepSeek.

## Структура проекта

```
VibeCode/
├── src/              # Исходный код приложения
│   ├── main.js       # Точка входа Electron
│   ├── preload.js    # IPC мост
│   ├── index.html    # UI структура
│   └── ui.js         # UI логика
├── ai/               # AI Router
│   └── router.js     # Маршрутизация AI запросов
├── agents/           # Агенты
│   └── selfdev.js    # SelfDev Agent
├── lib/               # Библиотеки
│   └── project-analyzer.js  # Анализ проектов
├── docs/              # Документация
│   ├── Vision.md      # Видение проекта
│   └── Roadmap.md     # План развития
└── config.json        # Конфигурация
```

## Импорт проектов из Cursor

VibeCode может анализировать проекты из Cursor:

1. Используйте кнопку "Анализ проекта"
2. Или перетащите проект в VibeCode
3. Система автоматически проанализирует структуру
4. AI сможет работать с проектом на основе его структуры

## Логи

Логи отображаются в боковой панели и автоматически обновляются каждые 2 секунды.

Логи также сохраняются в файлы в директории `/logs`.

## Конфигурация

Настройки в `config.json`:

- **AI провайдеры**: LM Studio, OpenRouter, GPT API
- **Smart Auto Mode**: автоматический выбор модели
- **SelfDev Agent**: пути к Vision и Roadmap

## Устранение проблем

### LM Studio недоступен

- Убедитесь, что LM Studio запущен
- Проверьте, что сервер слушает на порту 1234
- Проверьте, что модели загружены

### Ошибки генерации файлов

- Проверьте логи в боковой панели
- Убедитесь, что Vision.md и Roadmap.md существуют
- Проверьте права доступа к директории `/src`

### Falcon не отвечает

- Falcon работает только с английским языком
- Система автоматически переводит русские запросы
- При ошибках перевода используется DeepSeek
